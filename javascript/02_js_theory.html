Decision making, loops, and iteration — the logic engine of every program

MERN Stack — Module 2 if / switch / loops / array methods
01

if / else if / else

Basic Structure

const score = 72;

if (score >= 90) {
  console.log("Grade: A");
} else if (score >= 75) {
  console.log("Grade: B");
} else if (score >= 60) {
  console.log("Grade: C");  // ← this runs
} else {
  console.log("Grade: F");
}

Ternary Operator — One-Line if/else

Syntax: condition ? value_if_true : value_if_false

const age = 20;

// Long form
let access;
if (age >= 18) { access = "allowed"; }
else { access = "denied"; }

// Ternary — same thing
const access2 = age >= 18 ? "allowed" : "denied";

// In React JSX — you'll use this constantly
// <div>{isLoggedIn ? <Dashboard /> : <Login />}</div>

Common Mistakes

// ❌ Assignment inside condition — a nasty bug
let x = 5;
if (x = 10) { // = assigns, not compares! Always true.
  console.log("runs");  // always runs
}

// ✅ Should be
if (x === 10) { ... }

// ❌ Checking object/array equality by value
const a = [1,2];
const b = [1,2];
a === b;  // false — different objects in memory!
// Use JSON.stringify(a) === JSON.stringify(b) or a library

MERN context: In Express route handlers you'll check if (!req.body.email), in React you'll conditionally render components with ternary, in Mongoose you'll check if (!user) before returning 404.

02

switch Statement

Syntax & break

const day = "Monday";

switch (day) {
  case "Monday":
    console.log("Start of the week");
    break;                        // ← MUST have break or it falls through!
  case "Friday":
    console.log("End of work week");
    break;
  case "Saturday":
  case "Sunday":                 // two cases, one handler (fall-through used intentionally)
    console.log("Weekend!");
    break;
  default:                        // like else — runs if no case matches
    console.log("Midweek");
}

Use switch when

  • One variable, many fixed values
  • Matching strings or numbers
  • Redux reducers (action types)
  • HTTP method routing logic

Use if/else when

  • Range checks (> , <, >=)
  • Complex boolean conditions
  • Multiple variables in condition
  • Conditions with &&, ||
03

Loops

for Loop — Classic, Count-Based

// Syntax: for (init; condition; update)
for (let i = 0; i < 5; i++) {
  console.log(i);  // 0,1,2,3,4
}

// Loop over an array
const fruits = ["apple", "banana", "mango"];
for (let i = 0; i < fruits.length; i++) {
  console.log(fruits[i]);
}

for...of — Modern, Cleaner Array/String Loop

const fruits = ["apple", "banana", "mango"];

for (const fruit of fruits) {
  console.log(fruit);  // apple, banana, mango
}

// Also works on strings
for (const char of "MERN") {
  console.log(char);  // M, E, R, N
}

for...in — Loop Over Object Keys

const user = { name: "Ali", age: 28, role: "dev" };

for (const key in user) {
  console.log(`${key}: ${user[key]}`);
  // name: Ali
  // age: 28
  // role: dev
}
// Note: for...of on objects throws an error. for...in is for objects.

while Loop

let count = 0;
while (count < 3) {
  console.log(count);
  count++;
}
// 0, 1, 2
// Use when you don't know
// how many iterations upfront

do...while Loop

let n = 10;
do {
  console.log(n);
  n++;
} while (n < 3);
// prints 10 ONCE
// Body always runs at least
// once — then checks condition

break and continue

// break — exit the loop entirely
for (let i = 0; i < 10; i++) {
  if (i === 5) break;
  console.log(i);  // 0,1,2,3,4
}

// continue — skip this iteration, go to next
for (let i = 0; i < 6; i++) {
  if (i % 2 === 0) continue;  // skip even numbers
  console.log(i);  // 1,3,5
}
04

Array Iteration Methods — The MERN Workhorses

Quick Reference

.forEach()
Run a function for each item. No return value.
returns → undefined
.map()
Transform each item. Returns a NEW array.
returns → new array
.filter()
Keep items that pass a test. Returns NEW array.
returns → new array
.find()
Returns the FIRST item that passes the test.
returns → item or undefined
.findIndex()
Like find() but returns the index.
returns → number or -1
.some()
True if ANY item passes the test.
returns → boolean
.every()
True if ALL items pass the test.
returns → boolean
.reduce()
Boil array down to a single value.
returns → anything

forEach vs map — Most Confused Pair

const prices = [100, 200, 300];

// forEach — side effects only, returns nothing
prices.forEach((p) => console.log(p));  // just prints

// map — transforms, returns new array
const discounted = prices.map((p) => p * 0.9);
console.log(discounted);  // [90, 180, 270]
console.log(prices);      // [100, 200, 300] — ORIGINAL UNCHANGED

// In React you'll use .map() to render lists:
// products.map(p => <ProductCard key={p.id} product={p} />)

filter, find, some, every

const users = [
  { id: 1, name: "Ali",  active: true  },
  { id: 2, name: "Sara", active: false },
  { id: 3, name: "Riya", active: true  },
];

// filter — get all active users
const active = users.filter((u) => u.active);
// [{id:1,...}, {id:3,...}]

// find — first user with id 2
const sara = users.find((u) => u.id === 2);
// {id:2, name:"Sara", active:false}

// some — is any user inactive?
const hasInactive = users.some((u) => !u.active);  // true

// every — are all users active?
const allActive = users.every((u) => u.active);   // false

reduce — The Power Tool

const orders = [
  { item: "Book",  price: 250 },
  { item: "Pen",   price: 30  },
  { item: "Bag",   price: 800 },
];

// Sum all prices
const total = orders.reduce((acc, order) => {
  return acc + order.price;
}, 0);  // ← 0 is the initial accumulator value

console.log(total);  // 1080

// reduce can build any shape — array, object, number, string
// How it works:
// Round 1: acc=0,   order={Book,250}  → 0+250   = 250
// Round 2: acc=250, order={Pen,30}    → 250+30  = 280
// Round 3: acc=280, order={Bag,800}   → 280+800 = 1080

Chaining Methods — Real MERN Pattern

const products = [
  { name: "Laptop",  price: 45000, inStock: true  },
  { name: "Phone",   price: 20000, inStock: false },
  { name: "Tablet",  price: 30000, inStock: true  },
  { name: "Watch",   price: 8000,  inStock: true  },
];

// Get names of in-stock products priced above ₹10,000
const result = products
  .filter((p) => p.inStock && p.price > 10000)
  .map((p) => p.name);

console.log(result);  // ["Laptop", "Tablet"]

// This exact pattern is used in Express to process API results
// and in React to render filtered lists.

Rule of thumb: If you're writing a for loop over an array and building a new array inside it — stop. There's a method for that. map, filter, or reduce is almost always cleaner.
