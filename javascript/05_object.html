01

Objects Deep Dive — Creation, Methods, Shorthand

What Is an Object?

An object is a collection of key-value pairs where values can be any data type — including functions (called methods). Objects are the primary data structure in JavaScript and the shape of almost every piece of data you'll work with in MERN: a MongoDB document, an Express request, a React component's props — all objects.

Three Creation Patterns

// 1. Object Literal — most common, use this always
const user = {
  name: "Ahmed",
  age: 28,
  active: true,
  greet() {                          // method shorthand (ES6)
    return `Hello, I am ${this.name}`;
  }
};

// 2. Object.create() — sets prototype explicitly
const baseUser = { greet() { return "Hello"; } };
const admin = Object.create(baseUser);
admin.role = "admin";

// 3. Constructor Function / Factory (covered in Section 06)
function createUser(name, age) {
  return { name, age };   // shorthand — name:name, age:age
}

Property Shorthand, Computed Keys & Methods

const name = "Sara";
const age  = 25;
const key  = "role";

const user = {
  name,                     // shorthand for name: name
  age,                      // shorthand for age: age
  [key]: "admin",           // computed key → role: "admin"
  greet() {                 // method shorthand
    return `Hi ${this.name}`;
  },
  greetArrow: () => "Hi"    // arrow — no own this! (covered next)
};

// Dot vs Bracket notation
user.name;           // "Sara" — use when key is known
user["name"];        // "Sara" — same result
user[key];           // "admin" — use when key is a variable

// Optional chaining — safe access on deep/uncertain objects
user?.address?.city;  // undefined — no error even if address missing

Accessing, Adding, Updating & Deleting Properties

const car = { brand: "Toyota", year: 2020 };

car.color = "red";       // add new property
car.year  = 2023;         // update existing
delete car.color;         // remove property

// Check if key exists
"brand" in car;           // true
car.hasOwnProperty("year"); // true

MERN context: Every MongoDB document is a JS object. req.body, req.params, req.query in Express are all objects. React component props are objects. You'll be reading and writing object properties hundreds of times a day.

02

this in Objects — and Why It Breaks

this Inside a Method — Works Fine

const counter = {
  count: 0,
  increment() {
    this.count++;          // this = counter ✅
    return this.count;
  }
};
counter.increment();   // 1 — this correctly points to counter

Three Ways this Breaks

const user = {
  name: "Ali",
  greet() { return this.name; }
};

// Break 1 — method detached from object
const fn = user.greet;
fn();                    // undefined — this is now global/undefined

// Break 2 — passed as callback
setTimeout(user.greet, 100);  // undefined — setTimeout detaches it

// Break 3 — arrow function as method (no own this)
const obj = {
  name: "Sara",
  greet: () => this.name  // this = global scope, NOT obj
};
obj.greet();              // undefined

Rule: Always use regular function syntax method() {} for object methods that need this. Arrow functions do not have their own this — never use them as object methods.

03

call, bind, apply — Taking Full Control of this

The Problem They Solve

All three methods let you manually set what this points to when calling a function. Think of them as "function borrowing" — you take a method from one object and use it in the context of another.

call() — Invoke Immediately with Custom this

function introduce(city, country) {
  return `I am ${this.name} from ${city}, ${country}`;
}

const person1 = { name: "Ahmed" };
const person2 = { name: "Sara" };

// call(thisArg, arg1, arg2, ...)
introduce.call(person1, "Delhi", "India");
// "I am Ahmed from Delhi, India"

introduce.call(person2, "Dubai", "UAE");
// "I am Sara from Dubai, UAE"

// Real use case — borrowing Array methods for array-like objects
// (NodeList, arguments object — not real arrays)
function showArgs() {
  const arr = Array.prototype.slice.call(arguments);
  console.log(arr);
}
showArgs(1, 2, 3);   // [1, 2, 3]

apply() — Same as call(), but Arguments as Array

// apply(thisArg, [arg1, arg2, ...])  ← array instead of spread
introduce.apply(person1, ["Mumbai", "India"]);
// "I am Ahmed from Mumbai, India"

// Classic use — Math.max on an array (pre-spread syntax)
const nums = [3, 7, 1, 9, 2];
Math.max.apply(null, nums);   // 9  (null = no this needed)
// Modern equivalent: Math.max(...nums)

// Memory trick — A for Apply, A for Array

bind() — Returns a NEW Function with Locked this

function greet(greeting) {
  return `${greeting}, I am ${this.name}`;
}

const user = { name: "Priya" };

// bind does NOT call the function — returns a new one
const boundGreet = greet.bind(user);
boundGreet("Hello");     // "Hello, I am Priya"
boundGreet("Namaste");   // "Namaste, I am Priya"

// bind with pre-set arguments (partial application)
const helloGreet = greet.bind(user, "Hello");
helloGreet();             // "Hello, I am Priya"

// Fixing the broken callback from Section 02
setTimeout(user.greet.bind(user), 100); // ✅ this stays as user

Quick Comparison

MethodCalls immediately?ArgumentsReturnsUse when
call✅ YesComma-separatedResult of fnBorrow method once
apply✅ YesArrayResult of fnArgs already in array
bind❌ NoComma-separatedNew functionFix this for later use

MERN context: In Express, event listeners and middleware sometimes lose this context. In React class components, this.handleClick = this.handleClick.bind(this) in the constructor was the standard pattern before hooks. Understanding bind makes you debug these instantly.

04

Destructuring — Objects & Arrays

Object Destructuring

const user = { name: "Ahmed", age: 28, city: "Delhi" };

// Old way
const name = user.name;
const age  = user.age;

// Destructuring
const { name, age, city } = user;

// Rename while destructuring
const { name: fullName, age: userAge } = user;
console.log(fullName);   // "Ahmed"

// Default values
const { name, role = "viewer" } = user;
console.log(role);         // "viewer" (user has no role)

// Nested destructuring
const profile = { user: { name: "Sara", address: { city: "Mumbai" } } };
const { user: { name, address: { city } } } = profile;

// In function parameters — most common pattern in MERN
function createUser({ name, age, role = "user" }) {
  return { name, age, role };
}
createUser({ name: "Ali", age: 30 });  // { name:"Ali", age:30, role:"user" }

Array Destructuring

const colors = ["red", "green", "blue"];

// Position-based (unlike objects which are name-based)
const [first, second, third] = colors;

// Skip elements with commas
const [primary, , tertiary] = colors;  // skip green

// Default values
const [a, b, c, d = "yellow"] = colors;

// Swap variables — elegant trick
let x = 1, y = 2;
[x, y] = [y, x];   // x=2, y=1

// Rest in destructuring
const [head, ...tail] = [1, 2, 3, 4];
// head = 1, tail = [2,3,4]

// React useState returns an array — this is why!
const [count, setCount] = useState(0);  // array destructuring

MERN context: In Express — const { email, password } = req.body. In Mongoose — const { id } = req.params. In React — const { name, age } = props and const [state, setState] = useState(). You'll destructure hundreds of times daily.

05

Spread & Rest Operator

Spread — Expand an Iterable

// Arrays — spread into another array or function call
const a = [1, 2, 3];
const b = [4, 5, 6];
const combined = [...a, ...b];          // [1,2,3,4,5,6]
const copy     = [...a];               // shallow copy

Math.max(...a);                        // 3

// Objects — spread into another object
const defaults = { role: "user", active: true };
const newUser  = { ...defaults, name: "Ali" };
// { role:"user", active:true, name:"Ali" }

// Override — later keys win
const updated = { ...defaults, active: false };
// { role:"user", active:false }

// Clone + modify pattern — used constantly in React state updates
const user        = { name: "Sara", age: 25, city: "Delhi" };
const updatedUser = { ...user, age: 26 };  // changed age, rest intact

Rest — Collect the Remainder

// Rest in destructuring — collect what's left
const { name, password, ...safeData } = user;
// safeData = { age:25, city:"Delhi" } — password stripped out!
// This pattern is used in Node.js to remove sensitive fields before sending

const [first, ...rest] = [10, 20, 30, 40];
// first=10, rest=[20,30,40]

// Rest in function params — covered in Module 03
const sum = (...nums) => nums.reduce((a,b) => a+b, 0);

Spread ...

  • Expands/unpacks values outward
  • Used in assignments and calls
  • Creates shallow copies
  • Right side of = or in ()

Rest ...

  • Collects/gathers values inward
  • Used in destructuring & params
  • Must be last in the list
  • Left side of = or in params
06

Prototypes & the class Keyword

The Prototype Chain — Quick Recap

Every JS object has an internal [[Prototype]] link. When you access a property JS doesn't find on the object itself, it walks up the chain until it finds it or reaches null. This is why [1,2,3].map() works — map lives on Array.prototype, not on the array itself.

ES6 class — Clean Syntax Over Prototypes

class User {
  // constructor runs when: new User(...)
  constructor(name, email, role = "user") {
    this.name  = name;
    this.email = email;
    this.role  = role;
  }

  // Instance method — available on each instance
  greet() {
    return `Hi, I am ${this.name}`;
  }

  // Static method — called on the class, not instance
  static validate(email) {
    return email.includes("@");
  }
}

const u1 = new User("Ali", "ali@example.com");
u1.greet();              // "Hi, I am Ali"
User.validate("test@"); // true

Inheritance with extends

class Admin extends User {
  constructor(name, email, permissions) {
    super(name, email, "admin");  // call parent constructor
    this.permissions = permissions;
  }

  greet() {
    return `${super.greet()} [ADMIN]`;  // call parent method
  }
}

const admin = new Admin("Sara", "sara@x.com", ["delete", "ban"]);
admin.greet();   // "Hi, I am Sara [ADMIN]"

// instanceof checks
admin instanceof Admin;  // true
admin instanceof User;   // true  — Admin extends User

MERN context: Mongoose models are ES6 classes under the hood. When you write const User = mongoose.model('User', schema) and then new User({...}) — that's class instantiation. Custom error classes in Express also use extends Error.

07

Built-in Object Methods

The Essential Six

Object.keys(obj)
Array of all own enumerable keys
returns → string[]
Object.values(obj)
Array of all own enumerable values
returns → any[]
Object.entries(obj)
Array of [key, value] pairs
returns → [string, any][]
Object.assign(target, ...src)
Merge sources into target (mutates!)
returns → target
Object.freeze(obj)
Prevents all property changes
returns → frozen obj
Object.fromEntries(arr)
Convert [key,val] pairs → object
returns → object

In Practice

const user = { name: "Ali", age: 28, city: "Delhi" };

Object.keys(user);      // ["name", "age", "city"]
Object.values(user);    // ["Ali", 28, "Delhi"]
Object.entries(user);   // [["name","Ali"],["age",28],["city","Delhi"]]

// Loop over object — the modern way
for (const [key, val] of Object.entries(user)) {
  console.log(`${key}: ${val}`);
}

// Object.assign — merge (prefer spread for immutability)
const extra   = { role: "admin" };
const merged  = Object.assign({}, user, extra);  // new object
const merged2 = { ...user, ...extra };            // cleaner equivalent

// Object.freeze — immutable object (config, constants)
const CONFIG = Object.freeze({ PORT: 3000, DB: "mongodb" });
CONFIG.PORT = 8080;   // silently fails in normal mode, throws in strict

// Object.fromEntries — reverse of entries
const doubled = Object.fromEntries(
  Object.entries({ a: 1, b: 2 }).map(([k, v]) => [k, v * 2])
);
// { a: 2, b: 4 }
08

JSON — stringify, parse & Real API Patterns

What is JSON?

JSON (JavaScript Object Notation) is a text format for representing structured data. It looks like a JS object but has strict rules: keys must be double-quoted strings, no functions, no undefined, no comments. It is the universal data format for REST APIs.

JSON.stringify & JSON.parse

const user = { name: "Ahmed", age: 28, active: true };

// Object → JSON string (for API response / localStorage)
const jsonStr = JSON.stringify(user);
// '{"name":"Ahmed","age":28,"active":true}'

// Pretty print (for logging/debugging)
JSON.stringify(user, null, 2);
// {
//   "name": "Ahmed",
//   "age": 28,
//   "active": true
// }

// JSON string → Object (when you receive API data)
const parsed = JSON.parse(jsonStr);
parsed.name;   // "Ahmed"

// Things JSON drops silently
const obj = { fn: () => {}, undef: undefined, name: "Ali" };
JSON.stringify(obj);   // '{"name":"Ali"}' — fn and undef vanish!

Real MERN API Pattern

// Express — sending JSON response
app.get("/user/:id", async (req, res) => {
  const { id } = req.params;         // destructuring
  const user = await User.findById(id);
  if (!user) return res.status(404).json({ message: "Not found" });
  const { password, ...safeUser } = user.toObject();  // strip password
  res.json(safeUser);                // auto stringifies
});

// React — receiving JSON from API
const fetchUser = async (id) => {
  const res  = await fetch(`/api/user/${id}`);
  const data = await res.json();    // parse JSON → object
  const { name, email } = data;    // destructure
  return { name, email };
};

Deep clone trick: JSON.parse(JSON.stringify(obj)) creates a deep copy of an object. It loses functions and undefined values, but for plain data objects (like API responses) it is the simplest deep clone. For complex objects use structuredClone(obj) (modern JS) or the lodash library.
